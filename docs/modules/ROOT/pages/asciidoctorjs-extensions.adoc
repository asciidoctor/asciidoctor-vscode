= Asciidoctor.js extensions
:description: Load and register your own Asciidoctor.js extensions in the preview.

When rendering the preview, the extension can load and register your own Asciidoctor.js extensions.

== Where extensions live

By convention, extensions are located in `.asciidoctor/lib`, at the root of your workspace.
The extension recursively loads every `.js` file found there as an Asciidoctor.js extension.

For example, the following files are all loaded:

* `.asciidoctor/lib/emoji.js`
* `.asciidoctor/lib/emoji/index.js`
* `.asciidoctor/lib/foo/bar/baz.js`

== Enable extension registration

. In the extension settings, check *Enable Asciidoctor.js extensions registration* (`asciidoc.extensions.registerWorkspaceExtensions`).
. The first time, confirm that you trust the authors of the extensions located in `.asciidoctor/lib`.

image::asciidoctor-vscode-trust-exts.png[Asciidoctor.js extensions trust confirmation message]

[WARNING]
====
This feature executes JavaScript code.
Do not enable it unless you fully trust the authors of the Asciidoctor.js extensions in your workspace.
====

[TIP]
====
You can update the trust mode at any time with the *Manage Asciidoctor.js Extensions Trust Mode* command.
====

== Example: asciidoctor-emoji

You can write your own extension, or reuse an existing one.
Here is how to use the https://github.com/ggrossetie/asciidoctor-emoji[asciidoctor-emoji] extension:

. Install the npm package in the workspace directory:
+
[source,shell]
----
npm i asciidoctor-emoji
----

. Create `.asciidoctor/lib/emoji.js` with the following content:
+
[source,javascript]
----
module.exports = require('asciidoctor-emoji')
----

. Write an emoji, for example `:tada:`, and watch it render in the preview.

image::asciidoctor-vscode-emoji-ext.png[Asciidoctor.js Emoji extension enabled]

[NOTE]
====
`asciidoctor-emoji` does not embed any image.
It renders each emoji as an image served from the https://cdn.jsdelivr.net/npm/twemoji[Twemoji CDN] (for example `\https://cdn.jsdelivr.net/npm/twemoji@latest/2/svg/1f389.svg`), so an internet connection is required to display the emojis.

The preview's Content Security Policy allows remote images served over HTTPS by default, so these emojis are *not* blocked and you do not need to change any security setting.
If you ever need to load assets over plain HTTP instead, lower the preview security level with the *Manage Preview Security Settings* command.
====
