= Table Scroll Sync (issue #873)

// Manual test for https://github.com/asciidoctor/asciidoctor-vscode/issues/873
//
// Goal: verify that editor <-> preview scroll synchronization stays accurate
// across a single very large table that spans well beyond one viewport.
//
// How to test:
// 1. Open this file and the preview side by side.
// 2. Scroll the editor (wheel and scrollbar) through the middle of the table
//    and check that the preview follows to the matching row.
// 3. Scroll the preview and check that it does NOT snap back to the top of the
//    table, and that the editor follows without noticeable lag.
// 4. Watch the extension logs for "Illegal value for `line`" errors.
//
// The table below is intentionally long, uses spanned cells (.3+), multi-line
// cells, and empty cells so that source lines and rendered rows do not line up
// one-to-one -- the conditions that originally broke scroll sync.
