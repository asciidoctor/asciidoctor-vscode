// ============================================================================
// HOW TO USE
// ----------------------------------------------------------------------------
// This is an EDITOR feature, not a preview one: hover the mouse over an
// attribute reference `{name}` IN THIS EDITOR (no need to open the preview).
// A tooltip should pop up showing the resolved value.
//
// SCOPE (by design) — the hover resolves an attribute from the parsed document
// via `document.getAttributes()`. The extension parses with `parse: true` and
// never converts, so ONLY attributes known at the end of the HEADER are
// available:
//   - header attributes (declared before the first blank line),
//   - intrinsic attributes ({docname}, {docfile}, {docdir}, …),
//   - `.asciidoctorconfig` attributes,
//   - `asciidoc.preview.asciidoctorAttributes` settings.
//
// Attribute entries in the BODY (after the header) are only applied during
// CONVERSION, so they never appear in `getAttributes()`. See the KNOWN
// LIMITATIONS part below for what that excludes.
// ============================================================================

at a given position would require replaying attribute entries in document order
rather than reading the final state.
