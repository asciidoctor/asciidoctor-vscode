= Jump to a Remote Document Anchor — Manual Test Bench (#705)
:icons: font
:experimental:

// ============================================================================
// WHAT THIS FILE IS FOR
// ----------------------------------------------------------------------------
// Issue #705: "Jump to remote document anchor does not work".
//
// THE BUG
//   Clicking, in the PREVIEW, a link to another AsciiDoc file that ends with a
//   `#anchor` (e.g. `link:other.adoc#id[]` or `xref:other.adoc#id[]`) opened the
//   target document at its TOP instead of scrolling to the anchor. With the
//   default setting `asciidoc.preview.openLinksToAsciidocFiles = "inPreview"`,
//   the document was re-rendered in place but the `#fragment` was thrown away.
//
// THE FIX
//   The fragment is now carried into the freshly rendered document. On load the
//   preview scrolls to the element whose id matches the fragment and moves the
//   editor to its source line — exactly like clicking an in-page anchor.
//
// HOW TO RUN
//   1. Open this file, then "AsciiDoc: Open Preview to the Side".
//   2. Make sure `asciidoc.preview.openLinksToAsciidocFiles` is "inPreview"
//      (the default).
//   3. In the PREVIEW, click each link below in turn.
//      EXPECTED = the target document opens scrolled to the named anchor.
//      REGRESSION (old build) = the target opens at its title (top).
//   4. Use the editor "back" navigation (Alt+◀ / Ctrl+-) or reopen this file
//      to return between clicks.
// ============================================================================
